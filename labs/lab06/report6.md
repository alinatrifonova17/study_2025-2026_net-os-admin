## Выполнение лабораторной работы

Установка и настройка системы управления базами данных MariaDB

6.4.1 Установка MariaDB

В ходе выполнения лабораторной работы я выполнила установку и настройку системы управления базами данных MariaDB.

Сначала я установила необходимые пакеты для работы с базами данных. Для этого я выполнила следующую команду:

dnf -y install mariadb mariadb-server

После установки программного обеспечения я изучила конфигурационные файлы MariaDB, расположенные в каталоге /etc/my.cnf.d, а также основной конфигурационный файл /etc/my.cnf.

Файл /etc/my.cnf является основным конфигурационным файлом MariaDB и содержит основные параметры работы сервера базы данных.

Каталог /etc/my.cnf.d содержит дополнительные конфигурационные файлы, которые подключаются к основной конфигурации. В этих файлах могут задаваться параметры сервера базы данных, параметры клиента, настройки кодировки и другие параметры работы MariaDB.

После анализа конфигурационных файлов я запустила службу MariaDB и добавила её в автозагрузку системы.

Для этого я выполнила команды:

systemctl start mariadb
systemctl enable mariadb

После запуска сервера я проверила, что MariaDB прослушивает сетевой порт. Для этого я выполнила команду:

ss -tulpen | grep mysql

В результате выполнения команды я убедилась, что служба MariaDB успешно запущена и прослушивает соответствующий сетевой порт.

⸻

Настройка безопасности MariaDB

Следующим этапом я выполнила настройку безопасности MariaDB. Для этого я запустила специальный скрипт конфигурации безопасности:

mysql_secure_installation

В процессе выполнения скрипта я выполнила следующие действия:
	•	установила пароль для пользователя root базы данных;
	•	запретила удалённый доступ для пользователя root;
	•	удалила тестовую базу данных;
	•	удалила анонимных пользователей.

После завершения настройки безопасности я вошла в систему управления базами данных с правами администратора базы данных.

mysql -u root -p

После ввода команды система запросила пароль пользователя root базы данных.

Далее я просмотрела список доступных команд MySQL с помощью команды:

\h

Затем я вывела список баз данных, существующих в системе, используя SQL-запрос:

SHOW DATABASES;

В результате выполнения команды на экран был выведен список системных баз данных MariaDB. В стандартной установке обычно присутствуют следующие базы данных:
	•	information_schema
	•	mysql
	•	performance_schema
	•	test (может быть удалена при настройке безопасности)

После завершения работы с интерактивной оболочкой MariaDB я вышла из неё с помощью команды:

exit;


⸻

6.4.2 Конфигурация кодировки символов

На следующем этапе лабораторной работы я настроила кодировку символов MariaDB.

Сначала я снова вошла в систему управления базами данных с правами администратора:

mysql -u root -p

Далее я вывела статус MariaDB из интерактивной оболочки.

В выводе отображается информация о сервере базы данных, включая версию сервера, текущего пользователя, используемую кодировку, состояние соединения и другие параметры работы сервера.

После этого я вышла из оболочки MariaDB.

Далее я перешла в каталог конфигурационных файлов MariaDB:

cd /etc/my.cnf.d

В этом каталоге я создала новый конфигурационный файл для настройки кодировки:

touch utf8.cnf

После этого я открыла файл на редактирование и указала в нём следующую конфигурацию:

[client]
default-character-set = utf8

[mysqld]
character-set-server = utf8

Эти параметры задают использование кодировки UTF-8 как для клиента, так и для сервера базы данных.

После изменения конфигурации я перезапустила службу MariaDB:

systemctl restart mariadb

Затем я снова вошла в MariaDB и проверила статус сервера. В результате проверки я убедилась, что сервер теперь использует кодировку UTF-8.

6.4.3 Создание базы данных

На следующем этапе лабораторной работы я создала новую базу данных.

Сначала я вошла в MariaDB с правами администратора:

mysql -u root -p

Далее я создала базу данных с именем addressbook:

CREATE DATABASE addressbook CHARACTER SET utf8 COLLATE utf8_general_ci;

После создания базы данных я перешла к работе с ней:

USE addressbook;

Затем я проверила, какие таблицы присутствуют в базе данных:

SHOW TABLES;

На данном этапе база данных не содержала таблиц.

Далее я создала таблицу city со следующими полями:

CREATE TABLE city(name VARCHAR(40), city VARCHAR(40));

После создания таблицы я добавила в неё несколько записей.

INSERT INTO city(name,city) VALUES ('Иванов','Москва');
INSERT INTO city(name,city) VALUES ('Петров','Сочи');
INSERT INTO city(name,city) VALUES ('Сидоров','Дубна');

После добавления записей я выполнила запрос для просмотра всех данных таблицы:

SELECT * FROM city;

В результате выполнения запроса на экран была выведена таблица, содержащая три записи. В таблице отображаются фамилии людей и соответствующие им города.

⸻

Создание пользователя базы данных

Далее я создала отдельного пользователя для работы с базой данных.

CREATE USER user@'%' IDENTIFIED BY 'password';

После этого я выдала этому пользователю права на выполнение операций с базой данных addressbook.

GRANT SELECT, INSERT, UPDATE, DELETE ON addressbook.* TO user@'%';

После назначения прав я обновила привилегии базы данных:

FLUSH PRIVILEGES;

Далее я просмотрела структуру таблицы city:

DESCRIBE city;

В результате выполнения команды была выведена информация о столбцах таблицы, их типах данных и дополнительных параметрах.

После завершения работы я вышла из MariaDB:

quit


⸻


После выхода из MariaDB я просмотрела список баз данных:

mysqlshow -u root -p

Затем я просмотрела таблицы базы данных addressbook:

mysqlshow -u root -p addressbook

Также я проверила доступ пользователя user к базе данных:

mysqlshow -u user -p addressbook

В результате я убедилась, что пользователь имеет доступ к базе данных и может работать с её таблицами.

6.4.4 Резервные копии базы данных

На следующем этапе лабораторной работы я выполнила создание резервных копий базы данных.

Сначала я создала каталог для хранения резервных копий:

mkdir -p /var/backup


Затем я создала резервную копию базы данных addressbook:

mysqldump -u root -p addressbook > /var/backup/addressbook.sql


После этого я создала сжатую резервную копию базы данных:

mysqldump -u root -p addressbook | gzip > /var/backup/addressbook.sql.gz


После этого я выполнила восстановление базы данных из резервной копии:

mysql -u root -p addressbook < /var/backup/addressbook.sql


Также я выполнила восстановление базы данных из сжатой резервной копии:

zcat /var/backup/addressbook.sql.gz | mysql -u root -p addressbook

## Ответы на контрольные вопросы

1. Какая команда отвечает за настройки безопасности в MariaDB?
За настройку безопасности MariaDB отвечает команда mysql_secure_installation.

2. Как настроить MariaDB для доступа через сеть?
Для настройки сетевого доступа необходимо изменить параметры в конфигурационном файле /etc/my.cnf, например параметр bind-address, и разрешить подключение удалённых пользователей.

3. Какая команда позволяет получить обзор доступных баз данных после входа в MariaDB?
Для просмотра доступных баз данных используется команда SHOW DATABASES;.

4. Какая команда позволяет узнать, какие таблицы доступны в базе данных?
Для просмотра таблиц используется команда SHOW TABLES;.

5. Какая команда позволяет узнать, какие поля доступны в таблице?
Для просмотра структуры таблицы используется команда DESCRIBE имя_таблицы;.

6. Какая команда позволяет узнать, какие записи доступны в таблице?
Для просмотра записей используется команда SELECT * FROM имя_таблицы;.

7. Как удалить запись из таблицы?
Для удаления записи используется команда DELETE FROM имя_таблицы WHERE условие;.

8. Где расположены файлы конфигурации MariaDB?
Основные файлы конфигурации MariaDB расположены в /etc/my.cnf и в каталоге /etc/my.cnf.d.

9. Где располагаются файлы с базами данных MariaDB?
Файлы баз данных MariaDB обычно располагаются в каталоге /var/lib/mysql.

10. Как сделать резервную копию базы данных и затем её восстановить?
Для создания резервной копии используется команда mysqldump, а для восстановления базы данных используется команда mysql с указанием файла резервной копии.
