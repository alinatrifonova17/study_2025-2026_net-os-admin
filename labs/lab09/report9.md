Настройка POP3/IMAP сервера

## Цель работы

Целью лабораторной работы является получение практических навыков по установке и настройке почтового сервера для работы с протоколами POP3 и IMAP, а также изучение работы почтового сервера Dovecot и взаимодействия его с почтовым сервером Postfix.

⸻

## Последовательность выполнения работы

9.4.1 Установка Dovecot

В ходе выполнения лабораторной работы на виртуальной машине server я вошла в систему под своим пользователем и открыла терминал. Затем я перешла в режим суперпользователя.

sudo -i

После этого я установила необходимые пакеты для работы почтового сервера.

dnf -y install dovecot telnet

В результате выполнения команды в системе был установлен почтовый сервер Dovecot, а также утилита telnet, которая используется для тестирования сетевых служб.

⸻

9.4.2 Настройка Dovecot

Далее я приступила к настройке сервера Dovecot.

Сначала я открыла основной конфигурационный файл:

/etc/dovecot/dovecot.conf

В этом файле я указала список почтовых протоколов, которые должен поддерживать сервер.

protocols = imap pop3

Эта строка означает, что сервер будет работать с протоколами IMAP и POP3.

⸻

Далее я открыла файл настройки аутентификации:

/etc/dovecot/conf.d/10-auth.conf

В этом файле я убедилась, что используется метод аутентификации plain.

auth_mechanisms = plain

Данный метод означает, что пользователь передаёт логин и пароль для аутентификации на сервере.

⸻

После этого я открыла файл:

/etc/dovecot/conf.d/auth-system.conf.ext

В этом файле я проверила, что для поиска пользователей используется система PAM и файл passwd.

passdb {
    driver = pam
}

userdb {
    driver = passwd
}

Это означает, что сервер будет использовать системных пользователей Linux для аутентификации.

⸻

Далее я настроила расположение почтовых ящиков пользователей.

Я открыла файл:

/etc/dovecot/conf.d/10-mail.conf

В нём была указана следующая строка:

mail_location = maildir:~/Maildir

Это означает, что почтовые сообщения каждого пользователя будут храниться в каталоге Maildir в его домашней директории.

⸻

После этого я настроила сервер Postfix, указав каталог для доставки почты.

postconf -e 'home_mailbox = Maildir/'

Эта настройка указывает, что входящая почта должна сохраняться в каталоге Maildir пользователя.

⸻

Настройка межсетевого экрана

Далее я настроила межсетевой экран, разрешив работу служб POP3 и IMAP.

Сначала я посмотрела список доступных служб:

firewall-cmd --get-services

После этого я добавила необходимые службы.

firewall-cmd --add-service=pop3 --permanent
firewall-cmd --add-service=pop3s --permanent
firewall-cmd --add-service=imap --permanent
firewall-cmd --add-service=imaps --permanent

Затем я перезагрузила правила межсетевого экрана.

firewall-cmd --reload

После этого я проверила список активных служб.

firewall-cmd --list-services


⸻

Восстановление контекста SELinux

После изменения конфигурационных файлов я восстановила контекст безопасности SELinux.

restorecon -vR /etc


⸻

Запуск почтовых служб

После завершения настройки я перезапустила сервер Postfix и запустила сервер Dovecot.

systemctl restart postfix
systemctl enable dovecot
systemctl start dovecot


⸻

9.4.3 Проверка работы Dovecot

Для проверки работы почтовой службы я открыла дополнительный терминал и запустила мониторинг почтового журнала.

tail -f /var/log/maillog

Эта команда позволяет в реальном времени наблюдать за сообщениями почтового сервера.

⸻

Далее на сервере я просмотрела имеющуюся почту пользователя.

MAIL=~/Maildir mail


⸻

Для просмотра почтовых ящиков пользователя я использовала команду:

doveadm mailbox list -u user

Вместо user используется мой логин.

Эта команда выводит список почтовых ящиков пользователя.

⸻

Настройка почтового клиента

На виртуальной машине client я вошла под своим пользователем и перешла в режим суперпользователя.

Затем я установила почтовый клиент Evolution.

dnf -y install evolution

После установки я запустила программу Evolution и выполнила настройку почтового аккаунта.

В настройках учётной записи я указала:
	•	имя пользователя;
	•	адрес электронной почты в формате user@user.net;
	•	пароль пользователя.

Затем я перешла к ручной настройке.

Для входящих сообщений я указала сервер mail.user.net с протоколом IMAP и портом 143.

Для исходящих сообщений я указала сервер SMTP mail.user.net с портом 25.

В настройках безопасности для IMAP я указала STARTTLS и аутентификацию по обычному паролю.

Для SMTP была выбрана настройка без аутентификации.

После завершения настройки я отправила себе несколько тестовых сообщений и убедилась, что письма успешно доставляются.

Параллельно я наблюдала сообщения в журнале сервера с помощью команды tail, а также проверяла почтовые ящики с помощью команд mail и doveadm.

⸻

Проверка работы POP3 через Telnet

Для проверки работы протокола POP3 я использовала Telnet.

Сначала я подключилась к серверу.

telnet mail.user.net 110

После подключения я ввела имя пользователя и пароль.

user имя_пользователя
pass пароль

Далее я выполнила команды POP3.

Получение списка писем:

list

Получение первого письма:

retr 1

Удаление второго письма:

dele 2

Завершение сеанса:

quit

В результате выполнения этих команд я убедилась, что POP3-сервер работает корректно.

## Ответы на контрольные вопросы

1. За что отвечает протокол SMTP?
Протокол SMTP используется для отправки электронных писем между почтовыми серверами и почтовыми клиентами.

⸻

2. За что отвечает протокол IMAP?
IMAP используется для доступа к электронной почте на сервере и управления письмами непосредственно на сервере без их загрузки на компьютер пользователя.

⸻

3. За что отвечает протокол POP3?
POP3 используется для получения электронной почты с сервера на локальный компьютер пользователя.

⸻

4. В чём назначение Dovecot?
Dovecot — это сервер доступа к электронной почте, который обеспечивает работу протоколов POP3 и IMAP.

⸻

5. В каких файлах находятся настройки Dovecot?

Основные настройки находятся в каталоге:

/etc/dovecot/

Основные файлы:
	•	dovecot.conf — основной конфигурационный файл;
	•	10-auth.conf — настройки аутентификации пользователей;
	•	10-mail.conf — настройки хранения почты;
	•	auth-system.conf.ext — настройки источников пользователей.

⸻

6. В чём назначение Postfix?

Postfix — это почтовый сервер, который отвечает за отправку и доставку электронной почты между серверами.

⸻

7. Какие методы аутентификации используются в Dovecot?

Основные методы:
	•	plain — передача логина и пароля;
	•	login — простой механизм входа;
	•	CRAM-MD5 — более безопасная аутентификация с использованием хеширования.

⸻

8. Приведите примеры команд для работы с почтовыми протоколами через терминал.

Примеры:

telnet mail.server.net 110

USER username
PASS password

LIST

RETR 1


⸻

9. Приведите примеры работы с doveadm.

Просмотр почтовых ящиков пользователя:

doveadm mailbox list -u user

Просмотр информации о пользователе:

doveadm user user

Поиск сообщений:

doveadm search
